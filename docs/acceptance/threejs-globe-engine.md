# Three.js 地球引擎第四阶段验收

## 启动

```bash
npm install
npm run dev
```

打开浏览器访问默认开发地址。

## 验收项

- 页面加载后可看到一个可旋转、可缩放的地球
- 初始视角定位在亚洲区域附近
- 地球表面默认加载在线瓦片影像；网络失败时会降级到本地程序化影像
- 在线影像会随视角变化切换可见瓦片和 zoom，而不是启动时全量拉取全球固定 zoom 瓦片
- 地球表面优先加载真实高程位移；网络失败时保留程序化地形回退，而不是完全光滑球体
- 场景包含大气层光晕与星空背景
- 页面上至少存在 3 个可见点标记
- 页面上存在至少 1 条折线覆盖物
- 页面上存在至少 1 个填充多边形覆盖物
- 拖拽鼠标时地球会围绕地心旋转，松手后会按拖拽力度继续惯性转动并逐渐停下
- 滚轮缩放方向符合直觉：向下远离、向上靠近，且会按滚轮力度继续惯性缩放并逐渐停下
- 视角允许跨越北极或南极连续旋转
- 点击标记时右侧状态面板显示 `marker:<id>`
- 点击折线时右侧状态面板显示 `polyline:<id>`
- 点击多边形时右侧状态面板显示 `polygon:<id>`
- 点击普通球面时右侧状态面板显示经纬度
- 点击结果通过统一事件系统更新，而不是示例页自己做射线逻辑
- 窗口尺寸变化后画面仍能保持正确比例
- 相机与图层静止时不依赖持续渲染循环；惯性存在时才短时启用动画帧
- 在线瓦片采用缓存、调度器、视口 LOD 和可见区域裁剪，而不是无控制并发请求
- 生产构建产物包含异步 chunk，而不是所有 demo/runtime 都塞进主入口

## 自动验证
