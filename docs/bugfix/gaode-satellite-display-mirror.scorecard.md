# 缺陷修复质量评分卡 v2（100 分）

## 1) 评分维度

### A. 复现质量（0-25）

- 症状与影响范围明确：5
- 期望与实际清晰：5
- 复现环境明确：4
- 已稳定复现：5
- 触发条件明确：4

### B. 根因可信度（0-25）

- 直接原因明确：5
- 深层原因明确：5
- 证据链完整：5
- 根因可解释症状：5
- 漏检原因说明清楚：4

### C. 修复约束度（0-25）

- 最小改动：5
- 影响文件明确：5
- 风险点识别：4
- 防回归动作明确：5
- 回滚方案可执行：4

### D. 验证与发布准备（0-25）

- 冒烟验证完成：5
- 定向回归完成：5
- 全量门禁按需执行：5
- 结果总结明确：5
- 浏览器证据完整：3

## 2) 门禁阈值

- 轻量模式：总分 >= 75，且最小相关验证通过
- 标准模式：总分 >= 80，且已稳定复现或有等价失败测试
- 发布级模式：总分 >= 85，且回滚与 D0~D+2 观测完整

## 3) 评分记录模板

- 模式：`标准`
- A. 复现质量：23
- B. 根因可信度：24
- C. 修复约束度：23
- D. 验证与发布准备：23
- 总分：93
- 是否通过当前门禁：`是`
- 不通过时补齐项（owner + 截止时间）：`无`
