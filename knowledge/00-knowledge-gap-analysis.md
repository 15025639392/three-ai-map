## 3. 性能关键点标注

### 3.1 渲染性能关键点

```
⚠️ 关键性能瓶颈：

1. Draw Call 数量
   - 位置：FrameGraph.render()
   - 优化：实例化渲染、批处理、材质合并
   - 目标：< 1000 draw calls

2. 纹理切换次数
   - 位置：Material.bind()
   - 优化：Texture Atlas、材质排序
   - 目标：< 100 次/帧

3. 顶点数量
   - 位置：Geometry.update()
   - 优化：LOD、视锥裁剪、遮挡剔除
   - 目标：< 1M 顶点/帧

4. Shader 编译
   - 位置：首次渲染
   - 优化：预编译、缓存
   - 目标：启动时完成
```

### 3.2 内存性能关键点

```
⚠️ 关键内存瓶颈：

1. GPU 纹理内存
   - 位置：TextureCache
   - 优化：MIPMAP、压缩纹理、LRU 淘汰
   - 预算：256-512 MB

2. GPU 几何内存
   - 位置：GeometryCache
   - 优化：LOD、索引缓冲区、量化
   - 预算：128-256 MB

3. CPU 瓦片缓存
   - 位置：TileCache
   - 优化：LRU、字节预算、引用计数
   - 预算：100-200 MB

4. Worker 内存
   - 位置：WorkerPool
   - 优化：数据传输优化、共享内存
   - 预算：50-100 MB/Worker
```

### 3.3 网络性能关键点

```
⚠️ 关键网络瓶颈：

1. 并发请求数
   - 位置：RequestScheduler
   - 优化：全局并发限制、域名分片
   - 限制：6-8 并发/域名

2. 请求优先级
   - 位置：TileScheduler
   - 优化：可见性优先、距离优先、老化机制
   - 策略：critical > visible > prefetch

3. 缓存命中率
   - 位置：HTTP Cache / TileCache
   - 优化：缓存策略、预加载
   - 目标：> 80%
```

### 3.4 计算性能关键点

```
⚠️ 关键计算瓶颈：

1. SSE 计算
   - 位置：QuadtreeLOD.calculateSSE()
   - 优化：SIMD、缓存、增量计算
   - 频率：每帧/每瓦片

2. 坐标转换
   - 位置：CoordinateSystem
   - 优化：查表、批处理、Worker
   - 频率：每帧/每顶点

3. 碰撞检测
   - 位置：CollisionIndex
   - 优化：空间索引、增量更新
   - 频率：每帧/每符号

4. 几何生成
   - 位置：Bucket.addFeature()
   - 优化：Worker、增量生成、缓存
   - 频率：瓦片加载时
```
